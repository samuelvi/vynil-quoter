# Prompt para regenerar VinylQuoter con IA

Usa este prompt como instrucciones canónicas para reconstruir el proyecto desde cero. No trates `.opencode/`, `.opencode.local/`, `opencode-bundle/`, `memory-bank/` ni `.worktrees/` como parte del producto: son tooling externo o estado local.

```text
Actúa como agente builder senior y genera desde cero un proyecto llamado VinylQuoter.

Contexto inicial:
- El repositorio está vacío salvo instrucciones de producto.
- No hay código existente que conservar.
- El proyecto final debe ser una CLI en Go, ejecutada y probada mediante Docker y Makefile.
- No uses Python, Node ni scripts auxiliares para implementar el producto.

Objetivo del producto:
Crear una herramienta CLI que identifique discos de vinilo a partir de imágenes de portada y genere un catálogo CSV con valoración conservadora en euros.

Stack obligatorio:
- Go 1.23 desde el primer commit.
- Docker Compose en `docker/test` como runtime de aplicación y tests.
- Makefile como interfaz operativa principal.
- Todos los comandos de app y tests deben ejecutarse dentro de Docker.
- make run debe preparar Docker automáticamente.
- make test debe preparar Docker automáticamente.

Pipeline obligatorio:
1. Leer imágenes desde `data/src/`.
2. Preparar cada imagen en `data/dst/` antes de llamar al modelo.
3. Para formatos decodificables localmente, recortar localmente las imágenes decodificables de data/src, aislar la zona dominante de portada/vinilo y guardar JPG en `data/dst/`.
4. Para formatos soportados que no se puedan decodificar localmente, copiarlo a data/dst como fallback.
5. Enviar al proveedor de visión la ruta preparada de `data/dst/`, no la imagen original de `data/src/`; analizar la imagen preparada.
6. Identificar artista/banda, título de álbum, confianza, precio conservador en EUR, base de precio y notas.
7. Escribir o actualizar `data/report/album_catalog.csv`.

Directorios de datos:
- `data/src/`: imágenes de entrada.
- `data/dst/`: imágenes preparadas.
- `data/report/`: CSV final.
- `data/video/`: material audiovisual opcional.
- Ignora el contenido real de `data/` en Git y conserva solo `.gitkeep`.

Estructura Go objetivo:
- `go.mod`
- `cmd/vinyl-quoter/main.go`
- `internal/app`: parsing de flags, loop interactivo y orquestación.
- `internal/catalog`: CSV, cabecera, lectura, escritura, upsert y URLs de referencia.
- `internal/config`: valores por defecto, proveedores, modelos y calidades.
- `internal/crop`: preparación local de imágenes en `data/dst`.
- `internal/imageinput`: descubrimiento de imágenes soportadas.
- `internal/provider`: interfaz `Recognizer` y tipos comunes.
- `internal/provider/lmstudio`: cliente LM Studio compatible con API OpenAI.
- `internal/provider/gemini`: cliente Gemini.
- `internal/provider/visionpayload`: prompt y payload compartidos para visión.
- `internal/provider/jsonparse`: extracción robusta de JSON de respuestas de modelo.
- `internal/ui`: menú interactivo.
- `tests/`: tests Go que reflejen las rutas de implementación.

Formatos soportados:
`.jpg`, `.jpeg`, `.png`, `.webp`, `.dng`, `.heic`, `.heif`, `.tif`, `.tiff`.

Proveedores de reconocimiento:
- Por defecto: LM Studio local con modelo `qwen2.5-vl-7b-instruct`.
- Alternativo local: LM Studio con `gemma-3-4b-it`.
- Opcional: Gemini con `gemini-2.5-flash-lite`, usando `GEMINI_API_KEY` desde entorno.
- No imprimas secretos ni valores de variables de entorno.
- Los tests no deben hacer llamadas reales de red.

Menú principal requerido para `make run` sin flags:
0. `Salir`
1. `Procesar una imagen concreta`
2. `Procesar todas las imágenes de data/src`
3. `Guardado csv (<ruta actual>)`
4. `Modelo (<proveedor: modelo actual>)`
5. `Calidad carátula (<calidad actual>)`
6. `Calidad vinilo (<calidad actual>)`

Reglas del menú:
- Debe permanecer abierto después de cada acción.
- Solo debe salir cuando el usuario elige `Salir` y confirma la salida.
- La ruta CSV, proveedor/modelo y calidades seleccionadas deben persistir durante la sesión.
- `Guardado csv` permite cambiar ruta CSV, actualizar CSV, regenerar CSV desde cero o volver.
- El submenú Guardado csv no debe preguntar por modelo.
- `Modelo` permite elegir LM Studio por defecto, LM Studio alternativo o Gemini.

Defaults de configuración:
- Los valores iniciales del menú deben cargarse con esta precedencia: defaults hardcoded, `.env`, variables de entorno del proceso, flags CLI.
- El archivo `.env` usa formato `KEY=VALUE` con variables `VINYLQUOTER_*`.
- Debe existir `.env.example` versionado; `.env` y `.env.*` reales deben estar ignorados.

Condiciones/calidades:
- Calidad vinilo/media permitida: `M`, `NM/M-`, `VG+`, `VG`, `G+`, `G`, `F`, `P`.
- Calidad carátula/sleeve permitida: `M`, `NM/M-`, `VG+`, `VG`, `G+`, `G`, `F`, `P`, `Generic`.
- Ambas calidades deben defaultar a `VG`.
- Las calidades deben enviarse en el prompt del proveedor.
- El CSV debe guardar `condition` como `media: <grade>; sleeve: <grade>`.

Flags mínimos de CLI:
- `--src`: directorio de origen, default `data/src`.
- `--dst`: directorio de imágenes preparadas, default `data/dst`.
- `--report`: ruta CSV, default `data/report/album_catalog.csv`.
- `--image`: procesa una imagen concreta.
- `--all`: procesa todas las imágenes soportadas.
- `--replace`: regenera el CSV, solo relevante con `--all`.
- `--provider`: `lm-studio` o `gemini`.
- `--model`: override de modelo.
- `--lm-studio-base-url`: URL base compatible OpenAI.
- `--timeout`, `--max-retries`, `--retry-delay`.
- `--media-condition` y `--sleeve-condition`.

Makefile requerido:
- `make docker-build`
- `make docker-up`
- `make docker-down`
- `make docker-shell`
- `make build`
- `make run`
- `make run IMAGE=data/src/DSC01.jpg`
- `make run-all`
- `make run-all-replace`
- `make run-gemini`
- `make run-cli ARGS="..."`
- `make test`
- `make quality`
- Aliases compatibles: `make test-build`, `make test-up`, `make test-down`, `make test-shell`.

Comportamiento CSV:
- Cabecera exacta:
  `source_image,artist,title,identification_confidence,recommended_price_eur,condition,price_confidence,price_basis,notes,discogs_reference_url,ebay_reference_url,popsike_reference_url`
- `source_image` debe ser el nombre del archivo original, por ejemplo `DSC01.jpg`.
- El procesamiento normal debe reprocessar imágenes seleccionadas y hacer upsert por `source_image`.
- Las filas no seleccionadas deben conservarse.
- `--replace` con `--all` debe regenerar el CSV desde cero.
- Una ejecución de imagen única nunca debe reemplazar todo el CSV.
- `recommended_price_eur` debe contener solo números o rangos numéricos como `12` o `12-18`.
- Las URLs Discogs, eBay y Popsike deben ser búsquedas amplias basadas en artista/título, sin términos de condición como `VG+` o `sleeve`.

Documentación obligatoria:
- `README.md` debe ser ligero: qué es, flujo básico, comandos esenciales, enlaces a docs, layout y política de datos.
- `docs/QUICKSTART.md`: uso rápido.
- `docs/USAGE.md`: menú, flags, proveedores, condiciones y pipeline.
- `docs/CSV.md`: columnas, update/replace, precios, condiciones y URLs.
- `docs/DEVELOPMENT.md`: Docker, tests, calidad, estructura y Makefile.
- `docs/index.md`: índice enlazando toda la documentación mantenida.

Plan de ejecución obligatorio:
1. Restate objetivo, restricciones y criterios de aceptación.
2. Diseña arquitectura y fases antes de implementar.
3. Usa TDD siempre que sea viable.
4. Implementa en fases pequeñas y verificables.
5. Ejecuta tests dentro de Docker mediante Makefile.
6. No declares finalizado sin evidencia fresca de verificación.

Criterios de aceptación:
- `make test` pasa dentro de Docker.
- `make quality` pasa.
- `docker compose -f docker/test/docker-compose.yml config` es válido.
- `make run` abre el menú interactivo descrito.
- `make run-all` actualiza el CSV usando `data/src → data/dst → modelo → CSV`.
- `make run-all-replace` regenera el CSV desde los inputs actuales.
- La documentación principal enlaza a los documentos detallados.
```
